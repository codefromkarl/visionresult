# Visual Insight Agent - 多模态图片分析系统 PRD

## 目标

用户上传一张图片 → 系统自动分析 → 联网补充信息 → 输出结构化视觉调查报告

## 核心定位

**"视觉分析 + 证据链推理"**，而非 AI 聊天、通用 Agent 或单纯 OCR 工具。

## 输入

支持：手机照片、截图、社交媒体图片、游戏截图、街景、UI 界面

## 输出

结构化视觉调查报告，包含：

1. **场景分析** — 识别图片整体场景类型
2. **地点推测** — 基于视觉证据推断可能地点（附置信度）
3. **时间推测** — 推断拍摄时间段、季节、年份
4. **人物分析** — 人数、年龄段、行为描述
5. **OCR 内容** — 提取图中所有文字
6. **证据链** — 每个推断的依据列表

## 系统 Pipeline

```
图片输入 → 预处理 → 视觉分析层 → OCR 层 → 多模态理解层 → 实体抽取层 → 联网检索层 → 证据融合层 → 报告生成层
```

## 技术栈

### 后端
- Python 3.11+
- FastAPI（API 服务）
- LangGraph（多步骤 pipeline 编排）

### AI 模型
- **OCR**: PaddleOCR（中文/日文/UI 强）
- **VLM**: Qwen2-VL（本地部署）或 OpenAI/Gemini API
- **检测**: GroundingDINO（进阶方案）
- **分割**: SAM2（后期）

### 前端
- React + Next.js
- 支持上传/URL/剪贴板/拖拽

### 数据库
- PostgreSQL（任务/用户/图片元数据）
- pgvector 或 Milvus（向量数据库，相似图片搜索）
- Neo4j（后期，人物关系/地点关联/时间链）

### 图像处理
- OpenCV / Pillow（压缩、旋转、EXIF、清晰度增强）

## 模块设计

### 1. 图片输入模块
- 上传、URL、剪贴板、拖拽
- 前端：React + Next.js
- 后端：FastAPI

### 2. 图像预处理模块
- 压缩、旋转、EXIF 提取、清晰度增强
- 技术：OpenCV + Pillow
- 输出：宽高、EXIF 元数据

### 3. OCR 模块
- 核心：PaddleOCR
- 输出：文本 + bbox + 置信度
- 用途：地点推理的关键驱动（日文招牌、中文标识等）

### 4. 视觉检测模块
- 检测：人物、建筑、食物、Logo、UI、交通工具
- MVP：Qwen2-VL 直接分析
- 进阶：GroundingDINO + SAM2

### 5. 多模态理解模块（核心）
- 理解整个场景，而非仅识别物体
- Prompt 结构化输出：场景类型、地点推测、时间推测、人物行为、关键证据、不确定因素
- 输出格式：JSON

### 6. 实体抽取模块
- 自然语言 → 结构化信息
- 技术：spaCy + LLM 抽取
- 输出：地点关键词、品牌、建筑名等

### 7. 联网检索模块
- 检索来源：Google Maps、OpenStreetMap、Wikipedia、Google/Bing 搜索
- 后期：小红书、Instagram、Twitter/X
- 作用：将 OCR/VLM 识别的实体联网确认

### 8. 证据融合模块（真正核心）
- 融合：OCR、VLM、检索、EXIF、场景
- 策略："规则 + LLM" 混合，而非纯 LLM 决定
- 输出：概率 + 证据来源
- 决定系统是"AI 胡猜"还是"可信分析"

### 9. 报告生成模块
- 用户版：自然语言
- 专业版：结构化 JSON + Markdown

## 设计原则

1. **宁可不确定，也不要瞎猜** — 诚实表达不确定性
2. **证据链优先** — 每个推断必须有依据
3. **规则 + LLM 混合** — 不让 LLM 直接决定一切
4. **真正的产品壁垒是"谁的视觉证据链最可信"**

## MVP 阶段

### 第一阶段（2 周）
- 上传图片 → OCR → VLM 分析 → 输出报告
- 后端 FastAPI + PaddleOCR + Qwen2-VL/VLM API
- 前端 Next.js 上传界面

### 第二阶段
- 联网增强：识别实体 → 自动搜索 → 增强推理

### 第三阶段
- 证据链：地点概率 + 证据来源 + 置信度评估

## 技术难点

1. 防止幻觉
2. 提高可信度
3. 结构化推理
4. 多证据融合
